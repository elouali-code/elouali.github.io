Plateforme d'Entraide Étudiante : Architecture Microservices

Conception et développement d'une application distribuée scalable utilisant Spring Boot pour connecter les étudiants par compétences.

  • TypeProjet Académique
  • ArchitectureMicroservices (Distribuée)
  • BackendJava Spring Boot, Hibernate
  • CodeVoir sur GitHub

Le Contexte : Découpler pour mieux Scaler

L'objectif de ce projet était de sortir des architectures monolithiques classiques pour concevoir une plateforme d'entraide étudiante modulaire. Le défi technique principal résidait dans la gestion de la communication entre services indépendants et la cohérence des données dans un environnement distribué.

Architecture Technique

L'application est décomposée en services autonomes communiquant via API REST, chacun ayant sa propre responsabilité métier :

1. User Service (Port 8080)

C'est le référentiel des utilisateurs. Il gère le cycle de vie des étudiants (inscription, modification de profil) et surtout leurs compétences et les avis reçus. Il expose des endpoints consommés par les autres services pour valider l'identité ou récupérer des profils spécifiques.

2. Request Service (Port 8081)

Ce service centralise la gestion des demandes d'aide. Lorsqu'une demande est créée, il interroge le User Service de manière synchrone pour vérifier la validité du demandeur avant d'enregistrer la transaction.

3. Recommendation Service (Port 8082)

Le cœur intelligent de la plateforme. Il contient la logique de matching :

  • Il analyse le texte d'une demande d'aide.
  • Il interroge le User Service pour trouver les étudiants possédant les compétences requises.
  • Il suggère les meilleurs tuteurs potentiels et gère le système de notation (feedback) après l'aide.

Stack & Résultats

Spring Boot

Utilisation du framework standard de l'industrie pour la création rapide de microservices robustes.

Communication REST

Orchestration des flux de données entre services via des appels HTTP synchrones.

Persistance MySQL

Gestion des données relationnelles avec Hibernate pour l'abstraction (ORM) et la génération de schémas.

Frontend Découplé

Interface HTML/JS unique qui agrège et consomme les APIs des 3 services backend.
