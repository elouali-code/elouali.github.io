End-to-End Weather Data Pipeline

Conception d un pipeline ETL batch complet pour ingerer des donnees meteo depuis une API publique, les transformer, les modeliser en entrepot local et les visualiser dans un dashboard interactif.

RoleJunior Data Engineer
TypePersonal Project
NiveauProduction-Oriented
StackPython, Pandas, DuckDB, SQLite, Metabase

Description

Ce projet met en place un flux data engineering de bout en bout: ingestion API, stockage brut, transformation analytique, chargement en entrepot et exposition BI. L objectif est de reproduire un environnement local proche des pratiques de production.

Key Responsibilities

  • Built a Python data ingestion script to collect weather data from a public API.
  • Implemented raw data storage in JSON format (data lake layer).
  • Developed data transformation logic using Pandas.
  • Loaded curated data into a SQLite data warehouse.
  • Connected the warehouse to Metabase for business intelligence.
  • Designed interactive dashboards showing temperature and humidity trends by city.

Architecture Highlights

  • Batch ingestion pipeline.
  • Layered data architecture: raw -> transformed -> warehouse -> BI.
  • Reproducible local environment with virtualenv.
  • Modular and extensible pipeline design.

Tech Stack

  • Python, Pandas, DuckDB, SQLite.
  • Metabase, REST API, Data Modeling.

Business Value

This project demonstrates the ability to build production-style data pipelines, transform semi-structured data, model analytical datasets, and deliver actionable insights through BI dashboards.
