Projet : FREEZE (File REtention ZonE)

Conception et implémentation d'un service de stockage de fichiers intelligent, "serverless" et optimisé pour les coûts sur AWS.

  • RôleArchitecte Cloud / Développeur
  • TypeProjet Personnel
  • DuréeConcept & Développement
  • TechnologiesAWS Lambda, API Gateway, S3, DynamoDB, EventBridge, AWS CDK, Python
❮ ❯

Le Contexte : Un Stockage Intelligent et Économique

[cite_start]

Le défi était de concevoir un service de stockage (nommé "FREEZE") où les fichiers sont accessibles via une URI unique et immuable, même si leur emplacement physique change[cite: 4, 5]. [cite_start]L'objectif principal est de réduire les coûts de stockage en déplaçant automatiquement les fichiers entre différentes classes de stockage AWS (chaud, tiède, froid) en fonction de leur fréquence d'accès[cite: 2, 3]. [cite_start]Seul le service FREEZE lui-même peut effectuer des opérations d'écriture (création, suppression, déplacement) pour garantir l'intégrité du système[cite: 1].

Architecture & Fonctionnalités Clés

[cite_start]

La solution est entièrement basée sur une architecture "serverless" sur AWS, ce qui la rend hautement scalable, résiliente et rentable[cite: 7]. Elle s'articule autour de plusieurs services managés :

[cite_start]

Architecture "Serverless" sur AWS [cite: 38]

Le cœur du système repose sur des services qui ne nécessitent aucune gestion de serveur :

    [cite_start]
  • AWS Lambda : Exécute toute la logique métier : ajout, suppression, résolution d'URI et gestion du cycle de vie des fichiers[cite: 22, 111, 114].
    • [cite_start]
  • Amazon API Gateway : Sert de point d'entrée HTTP sécurisé pour toutes les interactions des clients avec le service[cite: 37].
    • [cite_start]
  • Amazon S3 : Assure le stockage physique des fichiers, en utilisant différentes classes (Standard, Infrequent Access, Glacier) pour l'optimisation[cite: 16, 17, 18].
    • [cite_start]
  • Amazon DynamoDB : Une base de données NoSQL qui stocke les métadonnées de chaque fichier (ID, URI S3, classe de stockage, date de dernier accès, compteur d'accès)[cite: 31, 36, 47].

Gestion Intelligente du Cycle de Vie des Fichiers

[cite_start]

Une fonction Lambda ("Storage Manager") est exécutée quotidiennement via une règle Amazon EventBridge pour analyser les métadonnées des fichiers[cite: 22, 417, 979]. [cite_start]Elle applique un algorithme hybride (combinant la récence LRU et la fréquence LFU) pour décider de "refroidir" ou de "réchauffer" un fichier[cite: 767, 844]:

    [cite_start]
  • Refroidissement (Cool-down) : Un fichier peu consulté (ex: moins de 5 accès, ou non accédé depuis 30 jours) est déplacé de S3 Standard vers S3 IA, puis vers Glacier[cite: 821, 822].
    • [cite_start]
  • Réchauffement (Warm-up) : Un fichier dans Glacier ou S3 IA qui redevient populaire (ex: accédé plus de 10 fois en 30 jours) est automatiquement ramené vers S3 Standard pour un accès plus rapide[cite: 824, 825].

Déploiement via Infrastructure-as-Code (IaC)

[cite_start]

L'ensemble de l'infrastructure est défini en Python à l'aide du AWS Cloud Development Kit (CDK)[cite: 768]. [cite_start]Cette approche garantit des déploiements fiables, reproductibles et automatisés, tout en supprimant les conflits potentiels, comme entre les règles de cycle de vie S3 natives et la logique Lambda personnalisée[cite: 998, 1004, 1082].

Résultats & Impact

Optimisation des Coûts

[cite_start]

Réduction significative des coûts de stockage grâce à la migration automatique et intelligente des fichiers vers les classes S3 les plus économiques[cite: 2, 3].

Haute Scalabilité

L'architecture "serverless" s'adapte automatiquement à la charge, gérant un grand volume de fichiers et de requêtes sans intervention manuelle.

Accès Simplifié et Stable

[cite_start]

Les clients bénéficient d'une URI unique et permanente pour chaque fichier, masquant toute la complexité de la gestion du stockage sous-jacent[cite: 4, 5].

Déploiement Automatisé (IaC)

[cite_start]

L'utilisation d'AWS CDK permet de déployer et de mettre à jour l'ensemble de l'infrastructure de manière fiable et reproductible[cite: 768, 903].
