Migration Architecture Serverless

Conception et implémentation d'une solution data moderne en O(1) pour résoudre les problèmes de performance et de scalabilité chez Airbus.

RôleArchitecte Cloud
EntrepriseAirbus
Tech StackAWS DynamoDB, Lambda, CDK

Le Défi : Latence & Scalabilité

L'architecture initiale reposait sur un stockage S3 statique (fichiers Parquet). Ce système imposait une complexité algorithmique de O(N) : pour lire ou écrire une seule donnée, il fallait parcourir tout le dataset. Résultat : une latence élevée et une incapacité à monter en charge.

Ma Solution : Architecture O(1)

J'ai conçu une architecture 100% Serverless pour garantir une complexité constante O(1), quelle que soit la taille des données.

  • Stockage : Migration vers Amazon DynamoDB pour des accès clé-valeur millimétrés.
  • Logique : Fonctions AWS Lambda pour traiter les lectures/écritures sans gestion de serveur.
  • Fiabilité : Utilisation de files SQS pour découpler les écritures et gérer les pics de charge.
  • Déploiement : Infrastructure as Code (IaC) complète avec AWS CDK (Python).

Résultats Clés

Performance Maximale

Latence réduite de plusieurs secondes à < 50ms (O(1)).

Coûts Optimisés

Modèle "Pay-as-you-go" éliminant les coûts de serveurs inactifs.

100% Automatisé

Déploiement IaC reproductible et sans erreur manuelle.
